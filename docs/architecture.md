# CoDevx — System Architecture

## Overview

CoDevx v4.0 is a **multi-process, event-driven system** composed of:

1. **`agent_mesh.py`** — FastAPI backend (Python 3.12, async)
2. **`command-center/`** — React 19 PWA (TypeScript, Tailwind, Vite)
3. **Messaging Layer** — Discord Bot | Twilio WhatsApp | ZeroClaw Gateway (choice of one)
4. **SQLite database** — `agent_mesh.db` (aiosqlite, async R/W)
5. **Git workspace** — `./workspace/` (code generated by agents is written here)

## Concurrency Model

- **FastAPI + uvicorn** run the event loop (single async thread by default)
- `execute_pipeline()` runs as `asyncio.create_task()` — non-blocking to new HTTP requests
- `asyncio.gather()` runs Frontend Dev + Backend Dev in parallel within each phase
- `_run_subprocess()` uses `asyncio.create_subprocess_exec()` — truly async subprocess, does not block event loop
- WebSocket broadcast (`_broadcast()`) is also `asyncio.create_task()` — non-blocking

## Database Schema

```sql
-- Append-only log ring buffer (max 100 kept in memory, all in DB)
CREATE TABLE IF NOT EXISTS logs (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    message    TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

-- Completed task history
CREATE TABLE IF NOT EXISTS task_history (
    id             TEXT PRIMARY KEY,
    description    TEXT,
    completed_at   TEXT,
    files_modified INTEGER,
    tests_passed   INTEGER,
    pr_url         TEXT,
    branch         TEXT,
    llm_used       INTEGER
);

-- Files generated per task (for /api/files/:taskId)
CREATE TABLE IF NOT EXISTS generated_files (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id    TEXT,
    path       TEXT,
    content    TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES task_history(id)
);

-- Agent memory (cross-task learning)
CREATE TABLE IF NOT EXISTS agent_memory (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id    TEXT,
    agent      TEXT,
    category   TEXT,    -- 'architecture' | 'patterns' | 'findings' | 'delivery'
    content    TEXT,    -- truncated to 600 chars
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

## Agent System Prompts (Summary)

Each agent has a detailed system prompt enforcing production standards:

| Agent | Prompt enforces |
|-------|-----------------|
| Architect | Structured doc: Overview / Files / APIs / Data Models / Dependencies / Security / Scalability |
| Frontend Dev | Named exports, no `any`, Zod, React Query, aria-labels, Tailwind dark mode, Error + Suspense boundaries |
| Backend Dev | Parameterized SQL, JWT auth, rate limiting via slowapi, structlog, httpx.AsyncClient, repository pattern |
| Database Engineer | IF NOT EXISTS, soft deletes, RLS, FK indexes, CHECK constraints, column comments |
| QA Engineer | ≥85% coverage target, mocked externals, pytest-asyncio, httpx.AsyncClient for integration tests |
| Security Analyst | Structured FINDINGS → PATCHES (full files) → VERDICT format, OWASP Top 10 checklist |
| DevOps Engineer | Multi-stage Dockerfile, pinned image versions, health checks, resource limits, CI lint→test→build→deploy |
| Project Manager | Fixed report structure: Delivered / Quality Gate / Deployment Status / Risks & Next Steps |

## File Parsing Protocol

All LLM outputs are parsed for file markers using `parse_files_from_llm()`:

```python
# Supported markers (case-insensitive):
# FILE: path/to/file.ext
# // FILE: path/to/file.tsx
# # FILE: path/to/file.py
# -- FILE: migrations/001_init.sql

# The parser extracts content between consecutive FILE markers
# Files are immediately written to GIT_WORKSPACE via _write_files_to_workspace()
```

## Security Architecture

| Layer | Mechanism |
|-------|-----------|
| Webhook authenticity | HMAC-SHA256 (`X-ZeroClaw-Signature` header) |
| Approval gating | Plan must be approved before `execute_pipeline()` runs |
| SQL injection | aiosqlite parameterized queries everywhere |
| Shell injection | `asyncio.create_subprocess_exec()` — no `shell=True` |
| Input validation | Pydantic v2 models on all webhook request bodies |
| GitHub tokens | Stored in `.env` only, never committed (`.gitignore`) |
| LLM cost guard | ZeroClaw sandbox: `max_cost_per_day = 5.00` USD |
| Rate limiting | ZeroClaw sandbox: `max_actions_per_hour = 100` |

## Scaling Considerations

- **Horizontal scaling:** FastAPI is stateless per request, but `SYSTEM_STATE` is in-process memory. For multi-instance scaling, replace with Redis pub/sub for WebSocket broadcasting.
- **SQLite concurrency:** aiosqlite handles single-writer async I/O well. For high-throughput production, migrate to PostgreSQL (swap aiosqlite for asyncpg, update queries).
- **LLM parallelism:** Frontend Dev + Backend Dev run in parallel via `asyncio.gather()`. Additional agents can be parallelized similarly once prompt dependencies allow.
- **Task queue:** Currently one pipeline runs at a time (guarded by `SYSTEM_STATE["current_task"]`). For multi-tenant use, add Celery + Redis task queue.
